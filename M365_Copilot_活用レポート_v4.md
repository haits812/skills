## 1. Microsoft 365 Copilotとは

### 製品の概要

Microsoft 365 Copilotは、OpenAI/AnthropicのLLM（大規模言語モデル）とMicrosoft Graph（組織内のメール・カレンダー・ドキュメント）を組み合わせて動作するAIアシスタント。

### 「副操縦士」という位置づけ

Copilotという名前には「副操縦士（Co-Pilot）」という意味がある。操縦士は利用する私たち人間であり、Copilotはあくまで操縦士を支援してくれる存在。完成品を一発で生成するのではなく、「たたき台」を作成し、それを人が確認・修正するという協働型のワークフローが基本となる。

### 対応アプリケーション

Microsoft 365 Copilotには、**Copilot Chat（独立したチャット）** と **各アプリ内のCopilot** の2つの使い方がある。

#### Copilot Chat（独立したチャット）

ブラウザやアプリで開く、ChatGPTのようなチャット画面。調べものや質問、組織データの検索などに使う。

- アクセス方法：[https://copilot.microsoft.com](https://copilot.microsoft.com) または Microsoft 365 Copilotアプリ
- 主な用途：情報検索、質問応答、組織データの横断検索、ファイルアップロードして質問

#### 各アプリ内のCopilot

Word、Excel、PowerPoint等のアプリ内で直接使えるCopilot機能。開いているファイルを対象に作業支援する。

| アプリ | 主な機能 |
|--------|----------|
| Teams | 会議の自動要約・議事録作成、アクションアイテム抽出 |
| Word | 文書作成・要約・書き換え、コーチング機能 |
| Excel | 数式補完、データ分析・グラフ作成 |
| Outlook | メール要約・返信ドラフト生成 |
| PowerPoint | スライド自動生成、翻訳（40言語対応） |
| OneNote / Loop / Forms / SharePoint | 各種支援機能 |

### 料金体系（2025年1月時点）

| プラン | 料金 | 備考 |
|--------|------|------|
| Microsoft 365 Copilot | 約4,000円/人/月 | M365契約が前提 |
| Copilot Chat（基本機能） | 無料 | Entraアカウント保有者 |
| Copilot Studio（基本利用） | M365 Copilotに含まれる | エージェントフロー・AI Builderは従量課金 |

## 2. 消費者向けCopilotとの違い

「Copilot」という名前のサービスは複数存在するため、混同しやすい。

| 製品名 | 対象 | 特徴 | 料金 |
|--------|------|------|------|
| **Microsoft Copilot**（旧Bing Chat） | 一般消費者 | Web検索ベースのチャットAI。bing.com/chatやWindows標準搭載 | 無料 |
| **Copilot Pro** | 個人ユーザー | 上記＋Office連携。個人向けM365契約者向け | 約3,200円/月 |
| **Microsoft 365 Copilot** | 法人 | **本レポートの対象**。組織データ連携、エンタープライズセキュリティ | 約4,000円/人/月 |
| **Copilot Chat** | 法人 | M365 Copilotの基本機能。**M365ライセンス保有者に追加料金なし** | M365ライセンス必須 |

### 重要な違い

- 消費者向けCopilotは**インターネット上の情報**を参照
- Microsoft 365 Copilotは**組織内のデータ（メール・ファイル・会議）** を参照
- 法人版はデータがAI学習に使用されない（エンタープライズデータ保護）

本レポートでは、法人向けの**Microsoft 365 Copilot**について解説する。

### Copilot ChatとMicrosoft 365 Copilotの違い

法人向けには「Copilot Chat（追加料金なし）」と「Microsoft 365 Copilot（有料）」の2つがある。**2025年8月〜10月のアップデートで追加料金なし版の機能が大幅に拡充された**ため、有料版を契約するかどうかの判断ポイントが変わっている。

**重要な前提：法人向け「追加料金なし」の正確な意味**
- **Copilot Chat（追加料金なし）**: Microsoft 365ライセンス（Business Basic/Standard/Premium、E3/E5等）が**前提**。つまり「**追加料金なし**」であって「**完全無料**」ではない
- **M365 Copilot（有料）**: 上記M365ライセンス + 約4,000円/人/月の追加料金

#### 2025年8月〜10月のアップデートで変わったこと

**追加料金なしのCopilot Chatでもできるようになったこと：**
- Word/Excel/PowerPoint/Outlook/OneNote内でCopilotを利用可能に
- 開いているファイルの要約・質問応答
- 基本的な文書作成支援
- **Agent Builderによるエージェント作成**（Webナレッジのみ）

**引き続き有料版（M365 Copilot）が必要なこと：**
- **Teams会議の議事録自動生成**（最も即効性の高い機能）
- **組織全体のデータを横断検索**（「〇〇プロジェクトの資料を探して」など）
- **高度なExcel分析**（Agent Mode、複雑なデータ処理）
- **Researcher/Analystエージェント**
- **Agent Builderで組織データ（SharePoint等）を使ったエージェント作成**（追加料金なしで利用可能）
- **Copilot Studioによる高度なエージェント開発**（対話型エージェントは追加料金なし）

#### 判断のポイント

**Copilot Chat（追加料金なし）で十分なケース：**
- 開いているファイルの要約や質問がメイン
- 基本的な文書作成・推敲支援で十分
- Teams会議の議事録は手動で対応できる
- Webナレッジのみを使った簡易的なエージェントを作りたい

**M365 Copilot（有料・約4,000円/人/月追加）が必要なケース：**
- **Teams会議の議事録を自動生成したい**（これが最大の差別化ポイント）
- **組織内の情報を横断検索したい**（「先月の営業会議の資料は？」など）
- **高度なデータ分析を行いたい**（Analyst、複雑なExcel処理）
- **組織データ（SharePoint等）を活用したエージェントを追加料金なしで作りたい**
- **Copilot Studioで高度なエージェントを開発したい**

#### コスト対効果の考え方

月額4,000円という金額は、月額サブスクリプションとして見ると高く感じるかもしれない。しかし、**人件費換算で考えると投資回収のハードルは極めて低い**。

**具体例1：投資回収に必要な最小時間削減**

月給50万円の社員を想定した場合：
- 月給：500,000円
- 月の労働時間：160時間（週40時間×4週間）
- **時給換算：3,125円/時間**
- M365 Copilot月額コスト：4,000円
- **回収に必要な時間削減：4,000円 ÷ 3,125円/時間 = 約1.3時間/月（78分）**

つまり、**月にたった1.3時間の業務時間を削減できれば元が取れる**。Teams会議の議事録作成だけでも、週1回1時間の会議があれば、議事録作成時間（30分）×4回 = 月2時間の削減が見込める。

**具体例2：実績ベースのROI試算（デンソーの事例）**

デンソーでは**月12時間の時間削減**を達成している（出典：[Microsoft Customer Stories - デンソー](https://www.microsoft.com/ja-jp/customers/story/19426-denso-corporation-microsoft-365-copilot)）。

- 月額投資：4,000円/人
- 時間削減効果：12時間/月
- 時給換算（月給50万円の場合）：3,125円/時間
- **削減効果の金銭価値：12時間 × 3,125円 = 37,500円/月**
- **投資回収率：37,500円 ÷ 4,000円 = 約938%（約9.4倍）**

この事例では、投資額の**約9倍以上の価値**を生み出している計算になる。

#### 実務での違いの例

| やりたいこと | Copilot Chat（追加料金なし） | M365 Copilot（有料・+約4,000円/人/月） |
|-------------|---------------------|---------------------|
| 「開いているWordの要点を教えて」 | ✅ できる | ✅ できる |
| 「この文章をフォーマルに書き換えて」 | ✅ できる | ✅ できる |
| 「会議の議事録を自動で作りたい」 | ❌ 自分でメモ → 後からチャットで整理 | ✅ Teams会議中に自動要約 → 議事録生成 |
| 「先月の〇〇プロジェクトの資料を探して」 | ❌ 自分でSharePointを検索 | ✅ Copilotが組織全体から検索 |
| 「このデータを深く分析して」 | △ 基本的な分析のみ | ✅ Agent Mode、高度な推論 |
| 「Web情報で答えるエージェントを作りたい」 | ✅ Agent Builderで作成可能 | ✅ Agent Builderで作成可能 |
| 「社内FAQエージェントを作りたい」 | ⚠️ 従量課金が必要 | ✅ Agent Builderで追加料金なし |
| 「高度なワークフロー型エージェント」 | ⚠️ Copilot Studioアクセス＋従量課金 | ✅ Copilot Studioで対話型は追加料金なし |

**結論：Teams会議の議事録自動生成と、組織データの横断検索が必要かどうか**が最大の判断ポイント。特にTeams会議の議事録自動生成は、M365 Copilot（有料・+約4,000円/人/月）でないと利用できない最も価値の高い機能。

**エージェント開発について：**
- **簡易的なエージェント**（WebナレッジでFAQ対応など）→ Copilot Chat（追加料金なし）のAgent Builderで可能
- **社内データを活用したエージェント**（SharePoint等）→ M365 Copilot（有料）なら追加料金なし、追加料金なし版は従量課金
- **高度なエージェント**（複雑なワークフロー、外部システム連携）→ Copilot Studio（管理者によるアクセス許可＋従量課金、またはM365 Copilot保有者は対話型エージェントが追加料金なし）

※機能は頻繁にアップデートされるため、最新情報は[Microsoft公式](https://www.microsoft.com/ja-jp/microsoft-365-copilot/pricing)を確認してください。

#### ブラウザ版（Web版）とデスクトップ版の違い

Copilot Chatの機能は、ブラウザ版とデスクトップ版で異なる場合がある。

| 観点 | ブラウザ版（Web版） | デスクトップ版 |
|------|-------------------|---------------|
| **新機能の提供** | 先行して提供される | 遅れて提供される場合あり |
| **更新チャネルの影響** | なし | 「半期エンタープライズチャネル」では機能制限あり |
| **管理者設定の影響** | 比較的少ない | ピン留め解除等で無効化される場合あり |

**推奨：新機能をいち早く使いたい場合はブラウザ版（Web版）を利用**

出典：[キーマンズネット - M365 Copilotの最新機能を早く使うなら「ブラウザ版」が良い理由](https://kn.itmedia.co.jp/kn/articles/2508/01/news006.html)

デスクトップ版でCopilotが表示されない場合は、以下を確認：
- 更新チャネルが「最新チャネル」または「月次エンタープライズチャネル」になっているか
- 管理者がCopilot Chatのピン留めを解除していないか
- M365サブスクリプションが対象ライセンスか

## 評価の視点：現時点での確実な効果 + 将来の拡張性

### 現時点で効果が出やすい領域（優先度：高）

**1. 会議支援（Teams）**
- 要約・議事録・アクションアイテム整理による負担軽減
- 最も即効性が高く、導入効果を実感しやすい

**2. 社内情報の活用**
- 情報探索時間の削減（組織横断的な活用に効果）

**3. データ分析（Excel）**
- 集計・分析・説明作成の支援
- データ整備状況により効果が変動

### 投資判断の目安と有料版の価値

**コスト**: 月額約4,000円/人

**削減時間の目安**: 月あたり1〜2時間程度の削減で投資回収可能

**追加料金なし版 vs 有料版（M365 Copilot）**
- 追加料金なし版：基本的な支援は可能
- **有料版で価値が明確になる機能**：
  - **会議支援**（議事録自動生成）
  - **組織内情報の活用**（横断検索）
  - **分析の深さ**（高度なExcel分析、Researcher/Analystエージェント）

### 今後の拡張余地と業務特化による上乗せ

**効果の上限が上がる可能性**
- ※管理者設定、情報整備等が前提
- 機能追加により適用範囲が拡大

**機能追加（伸びしろ）**
- 資料作成・支援の高度化で適用範囲が拡大

**業務特化（エージェント）**
- FAQ・手続き案内等を業務単位で再利用化
- Agent BuilderやCopilot Studioによる業務特化型エージェント

### 【重要】成功の鍵

**留意事項**: 効果は利用状況によるため、ユースケース共有等の**「定着施策」が不可欠**

**結論**: 現時点の会議支援等の効果に、将来の機能拡張が上乗せされる想定で評価する。

# 第2部：Microsoft 365 Copilotの活用シーンと実践例

## 4. こんな時に使える（アプリ別活用シーン）

このセクションでは、第1部のまとめで示した**「効果が出やすい領域」を中心に、実務での具体的な活用シーン**を紹介する。

### 【優先度：高】Teams（会議支援）

**なぜ優先度が高いのか**: 議事録作成の自動化で**即効性があり、導入効果を最も実感しやすい**機能。週1回1時間の会議があれば、議事録作成時間（30分）×4回 = 月2時間の削減が見込める。

#### 基本機能：会議要約・議事録自動生成

**前提条件**: 文字起こし（トランスクリプト）を有効化する必要あり

| シーン | Copilotの使い方 | 効果 |
|--------|-----------------|------|
| **会議後の議事録作成** | 会議終了後に要約を生成→Wordにエクスポート | **議事録作成時間を大幅短縮**（最も効果が高い） |
| 会議に遅刻した | 「これまでの内容は？」と質問 | 議論の流れを即座にキャッチアップ |
| 長いチャット履歴の確認 | 「このチャットの要点は？」と質問 | スレッドを読み返す時間を削減 |
| 会議室での対面会議 | Teams会議として接続しCopilotを有効化 | 対面会議でも自動議事録作成が可能 |
| アクションアイテムの抽出 | 「誰が何をすることになった？」 | タスク漏れを防止 |

**Tips**: 
- 会議の録画機能と組み合わせると、後から参照しやすい
- 要約は複数の形式（箇条書き、段落形式等）で生成可能
- 会議中にCopilot Chatで質問すれば、その場で疑問を解決できる

---

#### 【NEW】Facilitator（ファシリテーター）エージェント：会議進行の自動化

**Facilitatorとは**: Teams会議専用のAIエージェント。通常のCopilotが「会議後の要約」を作るのに対し、Facilitatorは**会議中の進行管理をリアルタイムで支援**する。

**主な機能**:

**1. リアルタイムメモの自動生成・共同編集**
- 会議中に自動でメモ（議事録）を生成
- Microsoft Loop連携で参加者全員が同時編集可能
- 会議が終わった瞬間に精度の高い議事録が完成

**2. 議題管理・進行状況の可視化**
- 会議予定に議題を設定しておくと、会議画面上部にアジェンダと進行状況を表示
- チャットで「@Facilitator 次の議題：〇〇（5分）、△△（10分）」と指示可能
- 議題ごとの時間配分を可視化し、タイマーで進行管理
- 議論が始まるとチェックマークで自動追跡

**3. 決定事項・未解決事項・アクションアイテムの自動抽出**
- 会議中の会話を分析し、主要な決定事項を自動抽出
- 未解決の質問（open questions）をリスト化
- アクションアイテムを自動検出し、担当者と期限を整理

**4. 会議チャットでの対話・支援**
- 会議中にチャットで「@Facilitator ここまでの議論をまとめて」と依頼可能
- 「@Facilitator 残り時間を教えて」で時間管理
- 参加者の質問にその場で回答（Web検索も活用）

**5. Planner連携でタスク作成**
- 「@Facilitator 〇〇さんを担当者としてタスクを作成して」で即座にタスク化
- 会議で決まった内容が「言いっぱなし」で終わらず、実行可能なタスクリストに

**6. Word/Loop文書の下書き作成**
- 「@Facilitator この議題についてWord文書を作成して」で下書き自動生成
- Loop連携で共同編集可能な文書を即座に作成

**活用例**:

```
会議開始時：
@Facilitator 議題は次の通りです。
・Q4振り返り（10分）
・来期戦略（20分）
・予算配分（15分）

会議中盤：
@Facilitator ここまでの決定事項を整理してください

会議終盤：
@Facilitator 残り時間を教えて
→「あと3分です。最後の議題に進みましょう」

会議終了後：
@Facilitator タスクをPlannerに追加してください
```

**Facilitatorの効果**:
- 司会者が進行管理に取られる時間を削減し、議論の内容に集中できる
- 議題漏れ・時間超過を防止
- 会議終了と同時に議事録・タスクリストが完成
- 参加者全員がリアルタイムでメモを確認・編集可能

**注意点**:
- M365 Copilotライセンスが必要
- 会議予定の「オプション」から事前に有効化が必要
- 会議の音声認識（トランスクリプト）がオンになっていることが前提
- チャットでの指示内容は参加者全員に見えるため、機密情報に注意

**通常のCopilotとFacilitatorの使い分け**:
- **会議後の振り返り・要約**: 通常のCopilot
- **会議中の進行管理・リアルタイム支援**: Facilitator
- **両方を組み合わせる**: 最も効果的（Facilitatorで進行管理 + 会議後にCopilotで深掘り分析）

### 【優先度：高】Copilot Chat - 職場モード（社内情報の活用・リサーチ）

**なぜ優先度が高いのか**: 組織内の**情報探索時間を大幅に削減**。「あのファイルどこだっけ？」「過去のプロジェクトの資料は？」といった横断的な情報検索に効果が高い。さらに、**Web検索と組織データを組み合わせた高度なリサーチ**も可能。

#### Copilot Chatの3つのモード

Copilot Chatには目的に応じた3つのモードがあり、画面上部で切り替え可能：

| モード | データソース | 主な用途 |
|--------|------------|----------|
| **Webモード** | インターネット上の情報（Bing検索） | 最新ニュース、業界動向、一般的な知識の検索 |
| **職場モード** | SharePoint、OneDrive、Teams、Outlook、Graph Connector等 | **組織内データの横断検索**、社内情報の活用 |
| **ノートブックモード** | 指定したファイル・ページ | **長文の詳細分析**、複数ファイルの統合分析 |

**職場モード**が組織データ活用の中心機能となる。

---

#### 職場モードでできること

**1. 社内資料の横断検索**

組織内のSharePoint、OneDrive、Teams、Outlookを横断的に検索。ファイル名を覚えていなくても、内容や関連プロジェクト名で検索可能。

| シーン | 使い方 | 効果 |
|--------|--------|------|
| **プロジェクト資料の検索** | 「〇〇プロジェクトの提案資料はどこ？」 | SharePoint/OneDrive内を横断検索 |
| **過去の議論の確認** | 「△△について過去に誰とどんな議論をした？」 | Teams/Outlookの履歴から検索 |
| **方針・規定の確認** | 「リモートワーク制度について教えて」 | 社内規定ドキュメントから回答 |
| **類似事例の検索** | 「過去の〇〇案件と似たケースはある？」 | 関連ファイル・メールを一括検索 |

**2. 人材・専門家の検索**

組織内の人材情報から、特定分野の専門家やプロジェクト経験者を検索。

| シーン | 使い方 |
|--------|--------|
| 専門家を探す | 「〇〇に詳しい人は誰？」 |
| 協業者を探す | 「△△プロジェクトに関わった人は？」 |
| 組織図の確認 | 「〇〇さんのマネージャーは誰？」 |

**3. 過去の活動履歴を検索して自己分析**

自分の過去のメール、Teams発言、作成ファイルから活動履歴を分析。

| シーン | 使い方 | 効果 |
|--------|--------|------|
| **業務振り返り** | 「今月私が関わったプロジェクトは？」 | 自動的に活動を集約・整理 |
| **成果の棚卸し** | 「Q3に私が作成した資料を全てリストアップして」 | 評価面談の準備に活用 |
| **時間配分の分析** | 「先月どの案件に最も時間を使った？」 | Teams/Outlook履歴から分析 |
| **コミュニケーション分析** | 「最近誰と最も頻繁にやりとりしている？」 | 協業関係の可視化 |

**4. Web検索と組織データの統合リサーチ**

職場モードでもWeb検索を併用可能（管理者設定による）。業界動向と社内データを組み合わせた高度なリサーチができる。

| シーン | 使い方 |
|--------|--------|
| **競合分析** | 「〇〇社の最新動向を調べて、うちの△△プロジェクトとの違いは？」 |
| **市場調査** | 「□□市場のトレンドを調べて、社内の関連資料もまとめて」 |
| **技術調査** | 「××技術の最新情報と、社内での活用事例を教えて」 |

**5. Context IQ：ファイルの直接参照**

プロンプト内で「/」を入力すると、OneDrive/SharePointのファイルを直接検索して参照できる。ファイルアップロード不要で、組織内ファイルを分析対象にできる。

```
例：
/ → ファイル検索画面が表示
「2024年度事業計画.pptx」を選択
「この資料の重点施策を3つ教えて」
```

#### ノートブックモード：深掘り分析

**ノートブックモードとは**: 複数のファイル・ページ・会議メモを1箇所に集約し、それらを横断的に分析するモード。長文の詳細分析に特化。

**職場モードとの違い**:
- **職場モード**: 組織全体から関連情報を自動検索
- **ノートブックモード**: 自分で選んだ特定のファイル群を深掘り分析

**活用シーン**:

| シーン | 使い方 |
|--------|--------|
| **複数資料の統合分析** | プロジェクト関連の5つのファイルをノートブックに追加し「全体の課題は？」 |
| **長文レポートの分析** | 100ページの報告書を追加し「重要ポイントを10個抽出して」 |
| **会議録の統合** | 過去3回の会議メモを追加し「未解決の課題をリストアップして」 |
| **プロジェクト専用Copilot** | 特定プロジェクトの資料を全て集約し、専用のAIアシスタントとして活用 |

**ノートブックの作成方法**:
1. microsoft365.com → 左側の「ノートブック」
2. 「新しいノートブック」を作成
3. Word、Excel、PowerPoint、PDF、OneNote、Loopファイルを追加
4. 追加したファイルを基にCopilotが回答

---

#### 活用のコツ

**1. 検索キーワードは具体的に**
- ✅ 良い例：「2024年Q3の営業会議の議事録」
- ❌ 悪い例：「会議の資料」

**2. 時期を指定する**
- 「先月の〇〇プロジェクトの資料」
- 「2024年に作成された△△関連のファイル」

**3. 見つかった資料はリンクで提示される**
- クリックすれば元ファイルにすぐアクセス可能
- 複数候補がある場合は一覧で表示

**4. Web検索の併用**
- 画面右上の「Web コンテンツ」トグルでオン/オフ切り替え
- 社内情報だけに絞りたい場合はオフに

### 【優先度：高】Excel - Agent Mode（データ分析・作成）

**なぜ優先度が高いのか**: **ゼロからのデータ作成・高度な分析・予測まで一気通貫でサポート**。複雑な数式を考える手間が削減され、データ整備状況により大きな効果が見込める。従来のCopilotを大幅に超える自律的なタスク実行が可能。

#### Agent Modeとは

Excel内のCopilotには「通常モード」と「Agent Mode」がある。**Agent Modeは、より高度なデータ分析と複数ステップの自動実行**に対応した新機能（2025年9月発表、12月Web版GA、2026年1月デスクトップ版展開開始）。

**通常モードとAgent Modeの違い**:

| 観点 | 通常モード | Agent Mode |
|------|-----------|------------|
| **タスクの実行** | 1つの指示に1つの応答 | 複数ステップを自動分解・実行 |
| **分析の深さ** | 数式作成、書式設定、簡単な集計 | **複雑な分析、データクリーニング、トレンド分析、予測分析** |
| **データ作成** | 既存データの編集のみ | **ゼロから表・データセットを作成可能** |
| **自律性** | 都度ユーザーが指示 | 計画→実行→検証を自律的にループ |

**Agent Modeの使い方**:
- **Web版Excel**: Copilot Chatを開き「エージェントモード」を選択
- **デスクトップ版**: 2026年1月中旬から順次利用可能に（環境により提供時期が異なる）

---

#### 活用シーン1：ゼロからデータ・表を作成

**従来の課題**: 表の構造を考え、データを入力し、書式を整える...という手作業が必要だった。

**Agent Modeで可能になること**: 自然言語の指示だけで、ゼロから実用的な表を自動生成。

| シーン | 指示例 | 生成される内容 |
|--------|--------|---------------|
| **チェックリスト作成** | 「旅行の持ち物チェックリストを作って」 | 項目一覧 + チェック欄（プルダウン）+ 条件付き書式 |
| **スケジュール表作成** | 「プロジェクトのガントチャートを作って」 | タスク一覧 + 期間 + 進捗バー |
| **仮想データ作成** | 「コーヒーショップの月次決算報告書を作成して」 | 売上・経費・利益の仮想データ + 計算式 |
| **フォーム作成** | 「休暇申請書をExcel方眼で作って」 | 承認欄・日付欄等を含む実用的な申請書 |
| **テンプレート作成** | 「営業日報のテンプレートを作って」 | 日付・顧客名・商談内容等の入力欄 |

**具体例**:
```
指示：「コーヒーショップの8月と9月の月次決算報告書を作成して」

生成内容：
- 売上、原価、経費、利益の仮想データ
- 月次比較表
- 前月比の増減分析
- 「フード併売率UPが必要」等の改善提案
```

#### 活用シーン2：高度なデータ分析

**従来の課題**: 相関分析、トレンド分析には統計知識とExcel関数の習熟が必要だった。

**Agent Modeで可能になること**: 自然言語で指示するだけで、統計的に正確な分析を自動実行。

| シーン | 指示例 | 実行内容 |
|--------|--------|----------|
| **相関分析** | 「動画の各指標の相関関係を分析して」 | 相関係数の計算 + ヒートマップ + 色分け |
| **トレンド分析** | 「売上をグラフにして最も高い月を見つけて、理由を分析して」 | グラフ作成 + ピーク検出 + 要因分析（自然言語で説明） |
| **比較分析** | 「CTRが高いもの10件、低いもの10件を比較して、タイトルの違いを分析して」 | 上位/下位抽出 + 特徴比較 + パターン分析 |
| **条件付き分析** | 「インプレッション1000以上のデータで再分析して」 | フィルタリング + 絞り込み分析 |
| **予測分析** | 「来月の売上を予測して」 | 過去データから予測モデル作成 + グラフ化 |

**具体例**:
```
指示：「この表から売上のトレンドを分析して最も高い月を見つけ、理由を説明して」

Agent Modeの実行内容：
1. データを自動抽出（998件を正確に処理）
2. 月別売上の集計
3. グラフ作成（折れ線グラフ）
4. ピーク月の特定
5. 要因分析（自然言語で説明）
   「8月が最も高い理由：夏季キャンペーン＋新商品投入の効果」
```

**高度な分析の実例**:
- 相関係数マトリクスを自動作成
- 相関関係の強さを色で可視化（赤=強い正の相関、青=強い負の相関）
- 同一項目（相関係数=1）を自動的にグレーアウト
- 統計的に有意な関係を自動検出して解説

#### 活用シーン3：データクリーニング・整備

**従来の課題**: 重複削除、欠損値処理、表記ゆれの修正には手作業が必要だった。

**Agent Modeで可能になること**: データ品質の自動チェックと修正。

| シーン | 指示例 |
|--------|--------|
| **重複削除** | 「重複データを削除して整理して」 |
| **欠損値処理** | 「空白セルを適切な値で埋めて」 |
| **表記ゆれ修正** | 「会社名の表記を統一して」 |
| **データ型変換** | 「日付列を正しい形式に変換して」 |

---

#### 活用シーン4：複雑な数式・書式設定

**従来の課題**: VLOOKUP、INDEX/MATCH、配列数式等の複雑な関数を覚える必要があった。

**Agent Modeで可能になること**: 目的を伝えるだけで、最適な関数を自動選択・設定。

| シーン | 指示例 |
|--------|--------|
| 複雑な集計式の作成 | 「部門別・月別の売上合計を計算して」 |
| 条件付き書式の設定 | 「目標未達の行を赤くハイライトして」 |
| 数式エラーの解決 | エラーセルを選択し「このエラーを解説して」 |
| グラフ作成 | 「このデータを見やすいグラフにして」 |

---

#### 活用シーン5：リサーチ結果をもとにテーブル作成

**Agent Modeの最大の強み**: 既存データの分析だけでなく、**リサーチ結果や知識をもとに新しい表を作成**できる。

| シーン | 指示例 | 実行内容 |
|--------|--------|----------|
| **市場調査データ作成** | 「日本の主要クラウドサービスの市場シェアを調べて表にして」 | Web検索 + データ整理 + 表作成 |
| **競合比較表作成** | 「主要CRMサービスの機能比較表を作って」 | 情報収集 + 比較軸設定 + 表作成 |
| **ベンチマークデータ作成** | 「SaaS企業の平均成長率を業界別にまとめて」 | 業界データ収集 + 集計表作成 |

#### Agent Mode活用のTips

**1. データは表形式（テーブル）に整理しておくと精度が上がる**
- Excel の「テーブル」機能で範囲を指定
- 列名を明確に設定

**2. 分析結果は自然言語で説明してもらえる**
- グラフだけでなく「なぜそうなったか」を文章で説明
- 経営層への報告資料として直接活用可能

**3. 複数ステップの作業も一度に指示できる**
```
良い例：
「このデータから重複を削除し、売上トップ10を抽出し、
グラフにして、特徴を分析してください」

→ Agent Modeが自動的にステップ分解して実行
```

**4. 途中で条件変更も可能**
```
「いや、売上基準は100万円以上で再分析して」
→ 条件を変更して再実行
```

**5. Python経験がなくても高度な分析が可能**
- 統計分析、機械学習的な予測もAgent Modeが自動実行
- プログラミング不要

## 5. エージェント活用ガイド

### エージェント活用の3段階：追加コストなしでどこまで使えるか

エージェント機能を理解する上で最も重要なのは、**「人間が話しかけて使う限り、月4,000円の範囲内で利用可能」**という点。以下の3段階で整理すると分かりやすい。

| レベル | 活用のカタチ | 起動のきっかけ | 月4,000円に含まれるか | 実現できること |
|--------|------------|---------------|---------------------|---------------|
| **Lv.1** | **汎用Copilot** | 人間の指示 | ✅ **含まれる** | メール要約、文書作成、Web検索 |
| **Lv.2** | **連携型エージェント** | 人間の指示 | ✅ **含まれる**（Fair Usage内） | SharePoint参照、FAQ対応、ナレッジ検索、プロンプト保存 |
| **Lv.3** | **自律型エージェント** | 時間・イベント | ❌ **従量課金** | 人間不在でも自動判断・実行（1回約40円〜） |

**💡 安心の境界線**

**Lv.1〜Lv.2（人間が話しかけて使う機能）は、月4,000円の範囲内で利用可能。**通常の業務利用であれば、追加料金を気にせず活用できる。

**Lv.3（自律型）は上級者向け**で、完全自動化を実現したい場合に検討する（後述）。まずはLv.1〜Lv.2の機能を使いこなすことが重要。

---

### エージェントとは

「エージェント」とは、**特定の目的に特化したAIアシスタント**のこと。通常のCopilotが汎用的な質問応答をするのに対し、エージェントは特定のタスクや業務領域に最適化されている。

| 概念 | 説明 | 例 |
|------|------|-----|
| **Copilot** | 汎用AIアシスタント | 「このメールを要約して」「資料を作って」 |
| **エージェント** | 特定目的に特化したAI | 「営業リードを育成するエージェント」「社内FAQ対応エージェント」 |

**補足：Excel Agent Modeとの関係**

第4章で紹介した**Excel Agent Mode**も広義の「エージェント」の一種だが、こちらは**Excelアプリ内での複雑なデータ分析・操作に特化したエージェント**として位置づけられる。Excel Agent Modeは自律的に計画→実行→検証を繰り返す点でエージェント的な性質を持つが、ここで説明する「エージェント」は**ユーザーが作成・カスタマイズできる対話型AIアシスタント**を指す。

---

### なぜエージェントが重要か

1. **専門性**: 汎用AIより特定タスクの精度が高い
2. **効率性**: 毎回同じ指示を入力する手間が省ける
3. **一貫性**: 同じ基準で処理されるため品質が安定
4. **拡張性**: 外部システム（Salesforce、ServiceNow等）と連携可能

### エージェントの種類

| 種類 | 説明 | 作成方法 | 難易度 | レベル |
|------|------|----------|--------|--------|
| **Microsoft製エージェント** | Microsoftが提供する汎用エージェント | 最初から利用可能 | ★☆☆ | Lv.1〜2 |
| **SharePointエージェント** | 特定のSharePointサイトの情報を参照 | SharePointサイトから数クリックで作成 | ★☆☆ | Lv.2 |
| **Agent Builderで作成** | プロンプトベースで作る簡易エージェント | Copilot内のAgent Builderで作成 | ★☆☆〜★★☆ | Lv.2 |
| **Copilot Studioで作成** | フロー・API連携を含む本格的なエージェント | Copilot Studioで開発 | ★★★ | Lv.2〜3 |
| **サードパーティエージェント** | ISVが提供するエージェント | エージェントストアから追加 | ★☆☆ | Lv.2 |

**💡 ポイント**：
- Lv.2までのエージェントは月4,000円に含まれる（Fair Usage Limit内）
- Lv.3（自律型）は従量課金
- Copilot Studioは利用可能だが、エージェントフロー・AI Builderツールは従量課金

### Agent Builderで作成：プロンプトベースの簡易エージェント

**Agent Builder**は、M365 Copilot内で**プロンプトを使ってノーコードでエージェントを作成できるツール**。よく使うプロンプトをエージェント化したり、特定の業務に特化したAIアシスタントを手軽に作成できる。

#### 作成手順

1. Microsoft 365 Copilotを開く
2. 「エージェント」→「新しいエージェントを作成」を選択
3. Agent Builderが起動
4. 以下を設定：
   - **名前**: エージェントの名前
   - **説明**: どんなエージェントか
   - **指示（プロンプト）**: エージェントの振る舞いを定義
   - **スタータープロンプト**: ユーザーが使いやすいように候補の質問を設定
5. 必要に応じて**ナレッジソース**を追加（SharePoint、OneDrive、Webサイト）
6. 必要に応じて**機能**を追加（コードインタープリター、画像生成）
7. テストして問題なければ「作成」

#### 活用例

| エージェント名 | 指示（プロンプト）内容 | ナレッジソース |
|---------------|---------------------|--------------|

| **議事録整形エージェント** | 「議事録を以下の形式に整形してください：参加者、議題、決定事項、アクションアイテム」 | なし |
| **製品FAQ対応エージェント** | 「製品に関する質問に、社内ナレッジをもとに回答してください。」 | 製品マニュアルのSharePointサイト |
| **営業提案書作成サポート** | 「過去の成功事例をもとに、顧客に合わせた提案書の構成を提案してください。」 | 過去の提案書フォルダ（OneDrive） |

#### Agent Builderのポイント

**メリット**:
- **ノーコードで作成可能**：プログラミング不要
- **テンプレートあり**：よくあるユースケースのテンプレートが用意されている
- **ナレッジソースを追加できる**：SharePoint、OneDrive、Webサイトを参照可能
- **すぐに使える**：作成後すぐにM365 Copilotで利用可能
- **7割の業務はこれで解決**：Power Automateフローなしでも、SharePoint参照＋生成回答で多くの業務が自動化できる

**制約**:
- **複雑なフロー・条件分岐は作れない**：単純な対話型のみ
- **外部API連携はできない**：Salesforce等との連携は不可
- **Power Automateフロー呼び出しはできない**：エージェントフローはCopilot Studio専用機能
- より高度な機能が必要な場合は**Copilot Studio**を使う（ただし従量課金）

### 【上級編】自律型エージェント：人間不在で動く完全自動化

**⚠️ 注意：この機能は従量課金（1トリガー約40円〜）が発生します。まずはLv.1〜Lv.2の機能を使いこなしてから検討してください。**

2025年以降、Copilotのエージェントは「指示を受けて動く」だけでなく、**自律的にタスクを実行**できるようになった。

#### 従来型との違い

| 観点 | 従来のエージェント（Lv.2） | 自律型エージェント（Lv.3） |
|------|-------------------|-------------------|
| 起動方法 | ユーザーの指示を待って動作 | トリガー（時間・イベント）で自動起動 |
| タスク範囲 | 単一タスクを実行 | 複数ステップを計画・実行・検証 |
| 人の介在 | 人が結果を確認して次へ | 必要に応じて人に確認を求める |
| **コスト** | **月4,000円に含まれる** | **従量課金（1トリガー約40円〜）** |

#### 具体例：Sales Development Agent

1. **トリガー**: CRMで新規リードが登録される
2. **計画**: リードの業種・規模を分析し、アプローチ方法を決定
3. **実行**: パーソナライズされたフォローアップメールを作成・送信
4. **検証**: 開封率・返信率をモニタリングし、次のアクションを判断
5. **確認依頼**: 商談化の可能性が高いリードは営業担当に通知

#### 利用可能な自律型エージェント（2026年1月時点）

| エージェント | 機能 | 状況 |
|-------------|------|------|
| Sales Development Agent | リード育成・フォローアップ自動化 | Frontier |
| Workflows Agent | スケジュール/イベントベースの自動化 | Frontier |
| Copilot Studio自律エージェント | 独立動作するカスタムエージェント構築 | 2025年6月GA |

**💡 自律型エージェントのまとめ**

自律型エージェントは強力だが、**導入前にLv.2（連携型エージェント）で十分な効果が出せないか検証することを推奨**。多くの業務は、人間が指示して使うエージェント（Agent Builder + SharePoint参照）で解決できる。

**自律型を検討すべきケース**：
- 夜間・週末に自動実行したいタスクがある
- 人間の判断を待たずに即座に対応が必要
- 大量の定型タスクを完全自動化したい

それ以外は、まず**Lv.2の機能を使いこなす**ことから始めよう。

### Copilot Studioで作成：フロー・API連携を含む本格的なエージェント

**⚠️ 検討前に確認：Agent Builderで解決できませんか？**

Copilot Studioは非常に強力なツールだが、**学習コストが高く、一部機能（エージェントフロー・AI Builder）は従量課金**となる。以下に該当しない場合は、まず**Agent Builder（Fair Usage内無料）で7割の業務を解決**してから検討することを推奨。

**補足**：M365 Copilotライセンスがあれば、Copilot Studioでエージェントを作成・公開することは可能。ただし、Power AutomateフローやAI Builder機能を使う場合は従量課金が発生する。

**Copilot Studioが必要なケース**:
- 外部システム（Salesforce、ServiceNow等）との連携が必須
- 承認フローなど、Power Automateとの統合が必要
- 複雑な条件分岐・ループ処理が必要
- 外部チャネル（Webサイト、WhatsApp等）への公開が必要

**Agent Builderで十分なケース**:
- SharePoint/OneDriveの情報を参照して回答
- よく使うプロンプトの保存・再利用
- 部署内FAQエージェント
- 社内ナレッジ検索

---

**Copilot Studio**は、**より高度なエージェント開発環境**。Agent Builderでは実現できない複雑な業務フロー、外部システムとのAPI連携、条件分岐処理などに対応できる。

**💡 ライセンスについて**

M365 Copilotライセンス（月4,000円）があれば、Copilot Studioでエージェントを作成・公開できる。ただし、以下の機能は**従量課金（Copilot Credit消費）**となる：

- **エージェントフロー**（Power Automateフロー実行）
- **AI Builderツール**（ドキュメント処理、画像認識等）
- **Fair Usage Limitを超える大量実行**

つまり、**基本的なエージェント作成は無料範囲内だが、高度な自動化機能を使うと従量課金が発生**する。

**向いているケース**:

- **単位**：Copilot Creditはエージェント利用量を測る共通通貨
- **料金**：$0.01/Credit（約1.5円/Credit）
- **購入方法**：
  - Pay-as-you-go（従量課金）
  - Copilot Studio license（月額$200で25,000 Credit）
  - Copilot Credit Pre-Purchase Plan（年間契約・ボリュームディスカウントあり）

#### Fair Usage Limitとは

**Fair Usage Limit（公正使用制限）**は、簡単に言えば**「普通の使い方なら気にしなくていい枠」**のこと。

- **具体的な数値は公開されていない**が、1日に数百回チャットするような極端な使い方をしない限り、追加料金は発生しない
- **通常の業務利用（1日10〜50回程度のエージェント実行）であれば心配不要**
- Microsoftは使用パターンの変化に応じてこの制限を更新する可能性があるが、現時点では「おかわり自由」に近い感覚で使える

**参考：Fair Usage Limitを超えそうな場合**

- 複雑なAI処理を1日中連続実行
- 大規模なデータ処理を自動化

このような極端なケースでは、Copilot Studio従量課金への移行を検討する必要があるが、**一般的な業務利用ではまず該当しない**。

## 6. 注意点・課題

### 情報整備が前提

AIが参照するデータが整理されていないと、古い情報や誤った情報をもとに成果物が作られる可能性。SharePointやOneDriveの情報整理が導入前の重要なステップ。

### 出力の確認は必要（ハルシネーション）

生成AIは時に、間違った情報をもっともらしく答えることがある（ハルシネーション）。これは現時点の生成AIでは避けられない制約であり、人の目による確認は避けられない。Copilotは「たたき台」を作成するものであり、最終的な品質保証は人間が行う必要がある。

### 定着施策が必須

導入しても現場で使われなければ投資が無駄になる。「社内で聞ける人がいる」「失敗しても大丈夫」という安心感を醸成し、人と仕組みの両面からのサポート体制を整えることが重要。

### コストの検討

月額約4,000円/人という価格は決して安価ではない。投資に見合う効果が得られるか、まずは一部の部署でスモールスタートし、具体的な業務での時間削減効果を測定した上で全社展開を検討することが現実的。

**正直なところ、「月4,000円を払う価値があるか」は人による**

個人的には、**「1人あたり月4,000円を払う価値があるか？」は、人によって感じ方が変わる**と思っている。

- **毎日のように会議・議事録・アンケートに追われる人** → 十分に安い投資
- **あまり使いどころがない人** → 正直ちょっと高い

つまり、**"使い方次第で神にも微妙にもなる"ツール**。だからこそ、全社一斉導入ではなく、効果が出やすい人・部署から段階的に広げていくのが現実的なアプローチだと考える。

## 7. 部署内で活用を広げるポイント

### ① Copilotの可能性を過小評価しない：DXと同じく「業務の可視化」が鍵

**LLMは「型」があれば巻き取る**

2025年から2026年にかけて、**本当にちゃんと型が作れている仕事は、LLMが巻き取ってくれる**時代になった。議事録作成、メール返信、データ集計、資料作成など、手順が明確な業務は相当数のタスク種別が自動化される可能性がある。

**「これはできないだろう」の半分はできる**

実際に使い込んでいくと、「これはさすがに無理だろう」と思っていたタスクの**半分くらいは実はできる**ことに気づく。過小評価せず、まずは試してみる姿勢が重要。

**ただしDXと同じ：可視化・マニュアル化が前提**

一方で、これはDXやRPAと全く同じ構造で、**結局、業務が可視化されていない、マニュアル化されていないと、AIを使おうが何だろうが自動化は難しい**。

- **型が作れている業務** → Copilotで劇的に効率化できる
- **型が曖昧な業務** → Copilotを使っても効果は限定的

**重要なのは「使いながら型を作る」アプローチ**

完璧な業務マニュアルを作ってから導入する必要はない。むしろ、**Copilotを使いながら「このタスクはどういう手順でやっているのか」を可視化していく**プロセスが有効。

1. まず小さなタスクでCopilotを使ってみる
2. うまくいったやり方（プロンプト・手順）を記録する
3. それを部署で共有し、横展開する
4. 繰り返すことで、徐々に「型」が見えてくる

つまり、Copilot導入は単なるツール導入ではなく、**「業務を見える化し、型を作っていく」業務改善プロジェクト**でもある。この視点を持つことが、成功の分かれ目になる。

---

### ② まずは自分が使ってみる

Copilotは「正解の型」があるツールではない。**1日1回はCopilotに話しかけてみる**、うまくいったことや失敗したことをメモしておく、といった小さな行動の積み重ねが、Copilotとの距離を縮める。

### ③ ユースケース集の横展開

「何にどう使ったらいいのかわからない」という声に対応するため、利用事例集を整備し、プロンプトと回答例をコピー＆ペーストで使える形で提供することが効果的。

### ④ 成功事例の共有

デンソー、東芝、住友商事など成功企業はすべてパイロット導入から開始。IT部門や有志から始め、成功事例を作ってから全社展開するアプローチが有効。部署内でも同様に、まず数人が使い込んで「これは便利」という事例を作ることが重要。

### ⑤ 段階的な教育プログラム

学情では3か月間6回のトレーニングを実施し、各回200〜250名が参加。アプリケーションごとの活用方法を実業務に即した具体的なユースケースで説明することが重要。

### ⑥ トップダウンの支援

JCBでは社長メッセージを社員向けポータルで発信し、週次Tips配信との相乗効果で利用率向上を実現。経営層や部署長のコミットメントが定着の鍵。

### ⑦ 試行錯誤を楽しむ文化

うまくいったことや失敗したことをチームで共有する。「こんな風に使ったらうまくいった」「これは期待はずれだった」という情報交換が、チーム全体のスキルアップにつながる。

## 8. リリースチャネルと今後の機能

Microsoft 365 Copilotは日々進化しており、新機能が次々と発表されている。**しかし、発表された機能が自動的に全員使えるようになるわけではない**。

**「新機能が使えない」理由は主に3つ：**

- **まだ正式リリース（GA）されていない**：開発中・プレビュー段階の機能
- **Frontierなどの早期アクセスプログラム**：管理者がオプトインしない限り使えない
- **管理者による有効化が必要**：セキュリティ・コンプライアンスの観点から意図的にオフにしている場合もある

つまり、「記事で見た新機能が使えない」「同僚は使えるのに自分は使えない」という場合、多くは**機能のリリース段階や組織の管理設定**の問題。

この章では、**「どんな機能が今後使えるようになるかもしれないか」を把握**してもらうことを目的としている。もし使いたい機能があれば、まずリリース段階を確認し、必要に応じて情報システム部門に相談してみると良い。

# 第2部のまとめ

Microsoft 365 Copilotは、Microsoft 365を業務基盤としている企業にとって、以下の理由から最適なAIアシスタントの選択肢：

1. **使い慣れたアプリ内で完結** — 新ツール学習コストゼロで即座に業務効率化を開始可能
2. **社内データとの連携** — SharePoint/OneDriveのファイルを自然に参照し、組織の文脈を理解した回答を生成
3. **エンタープライズセキュリティ** — 既存のM365セキュリティ設定を継承し、データがAI学習に使用されない
4. **Teams会議との統合** — 議事録作成の自動化で即効性があり、導入効果を実感しやすい
5. **拡張性** — Copilot Studioで業務特化のカスタムエージェント構築が可能

国内大手企業の事例が示すように、適切な導入・定着施策を講じることで、**月間10,000時間以上の業務時間削減、年間12億円規模のコスト削減**も実現可能。

**活用のポイント：**
- まずは自分が使ってみて、小さな成功体験を積み重ねる
- うまくいった事例を部署内で共有し、横展開する
- 「失敗しても大丈夫」という心理的安全性を醸成する
- 定着施策（ユースケース共有、相談できる体制）を整備する

### 【追加セクション】導入の失敗パターンと回避策

「導入したけれど、結局誰も使っていない」という事態を避けるための、現実的なチェックリストです。

#### 失敗パターン1：全社員への「一律配布」によるコスト爆死

-   **状況：** 期待感だけで全社員にライセンスを付与したが、実務でTeams会議や複雑なデータ集計をしない層が「たまにメールを要約するだけ」に使っている状態。
    
-   **本音：** 月4,000円のコストを「時短」で回収するには、月1.3時間以上の削減が必須。メールの要約数回では、正直コスパが悪いです。
    
-   **対策：** **「ヘビーデューティ層」からの段階導入。**
    
    -   週に3回以上会議がある、膨大なリサーチが必要、Excelで複数ソースの突合をする、といった「AIへの外注ニーズが高い層」を特定し、そこからライセンスを割り当てるのが現実的です。
        

#### 失敗パターン2：「丸投げ」による低品質アウトプット

-   **状況：** 「AIがやってくれる」と思い込み、プロンプト1行で資料を作らせて、中身をろくに確認せずに出力。結果、数字が間違っていたり、文脈がズレていたりして手戻りが発生。
    
-   **本音：** Copilotは「有能な新人」ではなく「24時間働く副操縦士」です。操縦士（人間）が目的地を指示し、計器をチェックしなければ墜落します。
    
-   **対策：** **「人間がレビューする」をフローに組み込む。**
    
    -   「Copilotが作った下書きを、人間が3分で検品する」というワークフローを徹底します。0から100を作らせるのではなく、AIに60まで作らせて、人間が80まで仕上げる感覚が最も効率的です。
        

#### 失敗パターン3：組織内の「情報がぐちゃぐちゃ」でAIが迷子

-   **状況：** SharePointやOneDriveに「最新\_v1」「最新\_v2」「2024案\_確定（コピー）」といったファイルが散乱。Copilotが古い情報を参照して回答してしまう。
    
-   **本音：** AIは魔法ではないので、ゴミ（古いデータ）を入れればゴミ（誤情報）が出てきます。
    
-   **対策：** **「参照先エージェント」の活用。**
    
    -   組織全体を検索させるのではなく、Agent Builderを使って「このフォルダの、この確定済みマニュアルだけを見て答えて」という専用エージェントを作ることで、精度の低さをカバーできます。
